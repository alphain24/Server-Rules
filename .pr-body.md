## Summary

Ships six new features and the bug fixes uncovered while testing them on a live MC 26.1.1 Fabric server.

## Features

- **Accept / Decline player-head buttons in the GUI.** Two `minecraft:player_head` items are rendered on row 3 of the rules menu ‚ÄĒ `alphain`'s head for `ACCEPT` (runs `rules accept`) and `_3RACHA`'s head for `DECLINE` (runs `rules decline`). Both use Inventory Menu's `"as_player": true` so the click dispatches through the player's own `CommandSourceStack`, which is required because `RulesCommand` calls `ctx.getSource().getPlayerOrException()`.
- **`/rules reload` re-opens the GUI and re-freezes affected players.** `RulesCommand.executeReloadAll` / `executeReloadPlayer` now delegate to `JoinHandler.onRulesReloaded`, which re-applies the freeze effects and re-opens the menu with a short `RELOAD_GUI_DELAY_MS = 200ms` delay.
- **Blindness effect fix.** The previous 40-tick duration (‚Čą2s) expired almost immediately. All three freeze effects (`SLOWNESS 255`, `JUMP_BOOST 128`, `BLINDNESS`) now share `FREEZE_DURATION_TICKS = 20 * 60 * 10` and are re-applied right before the menu opens in case something cleared them while the player was waiting.
- **`/rules-view` command.** New top-level `/rules-view` (plus a `/rules view` subcommand) opens the menu without applying freeze effects, so already-accepted players can re-read the rules whenever they want.
- **`pack.mcmeta` fix for MC 26.1.1.** The previous generator wrote `"pack_format": 101.1` (a float ‚ÄĒ Minecraft silently rejects the pack and parks it in `level.dat`'s `DataPacks.Disabled` list). The file now uses Mojang's 26.x schema with `[major, minor]` arrays for both fields, widened to `"min_format": [94, 0]` and `"max_format": [101, 1]` so the same datapack loads cleanly across the 1.21 ‚Üí 26.1.1 range. Every startup additionally refreshes `pack.mcmeta` and calls `reloadResources(...)` with `file/Server Rules` re-selected, which auto-heals worlds whose pack had been disabled by the old float value.
- **Smooth 3-second GUI appearance on join.** Freeze + blindness are applied immediately on join (so the screen fades to black smoothly and the player can't move), and the menu is scheduled on the server thread `JOIN_GUI_DELAY_MS = 3000ms` later.

## Additional fixes from in-server testing

- **Legacy `rules.json` auto-upgrade.** Worlds upgraded from the previous build kept an old `rules.json` without the accept/decline buttons. The generator now force-rewrites the file when either `"rules accept"` or `"rules decline"` is missing. Admins who hand-edit the menu can drop an empty `.customized` marker next to `rules.json` to opt out of future auto-upgrades.
- **Custom-name and lore as JSON text components.** All `custom_name` and `lore` fields are now proper text components (`{"text":"‚Ä¶","color":"green","bold":true,"italic":false}`) with `"italic": false` to avoid the default italic styling Minecraft applies to renamed items under the stricter `ComponentSerialization.CODEC` in 26.1.

## Files changed

- `src/main/java/in/alphain/serverrules/ServerRules.java` ‚ÄĒ new `SERVER_STARTED` hook that re-scans the pack repository and re-enables `file/Server Rules` via `reloadResources(...)`.
- `src/main/java/in/alphain/serverrules/datapack/DatapackGenerator.java` ‚ÄĒ pack-format range, player-head buttons with `as_player`/`silent`, text-component labels, legacy-file detector, `getPackId()` helper.
- `src/main/java/in/alphain/serverrules/datapack/RulesGuiOpener.java` ‚ÄĒ 10-minute freeze duration shared across slowness/jump-boost/blindness.
- `src/main/java/in/alphain/serverrules/handler/JoinHandler.java` ‚ÄĒ `beginRulesFlow` with 3s schedule for joins and 200ms schedule for reloads.
- `src/main/java/in/alphain/serverrules/command/RulesCommand.java` ‚ÄĒ `/rules-view` top-level command, `view` subcommand, reload dispatches through `onRulesReloaded`.
